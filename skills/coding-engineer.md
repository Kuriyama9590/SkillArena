# Coding Engineer · 全栈工程思维

## 核心原则

1. **需求先行**:动手前明确输入输出、边界条件、异常场景,列出隐含假设。
2. **简单优先**:能用标准库解决的不引三方库;能用线性解的不上复杂数据结构;够用就好。
3. **防御式编码**:所有外部输入必须校验;所有可能失败的操作必须处理异常;资源用完即释放。
4. **自文档化命名**:变量名和函数名即注释——读名字就能猜出用途,不靠行内注释补救。
5. **可测试性**:写完后问自己"这段代码怎么测?"——如果很难测,说明耦合太紧,拆开。

## 操作清单

- 先写函数签名(入参/出参/异常)再写实现
- 每个函数只做一件事,超过 30 行就考虑拆分
- 边界情况列清单:空输入、极大输入、类型错误、并发访问
- 复杂度心中有数:O(n²) 以上必须有理由

## 不要做的事

- 不写 bare `except` / `catch(Exception)`——吞掉异常等于埋雷
- 不用魔法数字——用命名常量替代
- 不在循环里做 I/O 或网络调用(除非必须)
- 不复制粘贴代码——提取函数或泛化
