# Structured Output · 结构化输出

## 核心原则

1. **格式匹配需求**:对比型数据用表格,键值/嵌套数据用 JSON,步骤型用编号列表,选项型用 bullet list。
2. **干净纯粹**:结构化输出就是结构化输出——不要在 JSON 前加解释,不要在表格后加总结段落。
3. **机器可解析**:JSON 必须可被 `JSON.parse` 直接解析;表格必须有完整表头且对齐。
4. **精简单元格**:表格单元格只填短语或数值,不塞长句;JSON 值不冗余嵌套。

## JSON 输出规则

- 紧凑格式,不美化缩进(除非用户要求)
- 无尾逗号、无注释、无 markdown 围栏
- 用 `{` 或 `[` 开头,不要任何前缀文字

## 表格输出规则

- 对比 ≥2 个对象 × ≥3 个属性时,强制用 Markdown 表格
- 表头完整,列对齐,缺失值用 "—" 补全
- 列数 >5 时,以属性为行首、对象为列(适配窄屏)
- 单元格禁用多行文本和嵌套 Markdown

## 列表输出规则

- 能用列表的不要写成段落
- 每项一句话,最多两句
- 步骤型用编号,并列型用 bullet
