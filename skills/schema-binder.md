# Schema Binder · 结构化模式绑定

## 核心原则

1. **模式即契约**:用户给出 schema(字段名/类型/必填/枚举)时,输出必须 100% 契合,不多不少。
2. **纯净载荷**:只输出 schema 对应的数据本身,无 markdown 围栏、无解释、无前后缀文字。
3. **类型严格**:字符串不加引号外的内容,数字不用字符串伪装,布尔只有 true/false,空值用 null 而非空串。
4. **枚举收敛**:字段值若是枚举,只取允许值之一,绝不发明新值。
5. **缺字段处理**:无法填充的必填字段用 null 或空字符串占位,而非整条删除或编造。

## 操作清单

- 输出前在内部把目标 JSON 结构画一遍,逐字段对应
- 校验:每个键名拼写/大小写与 schema 完全一致
- 校验:数组长度、嵌套层级、必填项齐全
- 校验:无尾逗号、无注释、可直接被 `JSON.parse` 解析
- 若 schema 有示例,以示例为类型推断基准

## 不要做的事

- 在 JSON 外面包 ```json 围栏或加"以下是结果:"
- 把 schema 里没有的字段自作主张加上去"更完整"
- 用中文标点混入(全角逗号、中文引号)
- 数组里元素结构不一致(前两个对象、后一个字符串)
