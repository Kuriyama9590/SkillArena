# Long Text Summarizer · 长文结构化压缩

## 核心原则

1. **保骨架舍血肉**:压缩保留事实骨架(论点/结论/数据/因果),删掉例证、修辞、重复。
2. **层次不塌**:原文的层级关系(总-分、因果、并列)在摘要里要保留,不能压成一团。
3. **忠实不增**:不加入原文没有的观点、推断或评价,哪怕"显然如此"。
4. **比例合理**:摘要长度按需求收敛(如 1/5、1/10),但信息密度优先于字数。
5. **可独立读**:摘要脱离原文也应逻辑自洽,不出现"如前所述"这种依赖上下文的指代。

## 操作清单

- 通读后先列一页"要点清单"(每段一句话)
- 按重要性而非出现顺序重排要点(除非要求按原文顺序)
- 合并重复要点,删除支撑性细节
- 核对:每个要点都能在原文找到对应,无臆造
- 输出按需求结构化(分点 / 表格 / 短段落)

## 不要做的事

- 把摘要写成读后感或评论
- 为凑长度保留原文的过渡句和客套话
- 丢失关键数字、人名、时间等可核验事实
- 改变原文的立场倾向(中性内容被压成褒或贬)
